# pi-mono SDK Upgrade 0.63.1 → 0.67.6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade `@mariozechner/pi-coding-agent` (and companion packages) from 0.63.1 to 0.67.6, fix the one breaking API change, adopt `defineTool()` for cleaner custom tool definitions, and rebuild the agent-worker Docker image.

**Architecture:** The upgrade touches only `agent-worker/` — Go API and MCP Gateway are unaffected. The only mandatory code fix is `new ModelRegistry(authStorage)` → `ModelRegistry.inMemory(authStorage)` (0.64.0 breaking change). The `defineTool()` adoption (0.65.0) removes the `as unknown as ToolDefinition` cast in agent-runner.ts and tightens type safety in gateway-tools.ts.

**Tech Stack:** Node.js 22+, TypeScript, `@mariozechner/pi-coding-agent` 0.67.6, npm

---

## Version Gap Summary (0.63.1 → 0.67.6)

### Breaking changes that affect akmatori

| Version | Change | File to fix |
|---------|--------|-------------|
| 0.64.0 | `new ModelRegistry(authStorage)` constructor removed; use `ModelRegistry.inMemory(authStorage)` | `agent-worker/src/agent-runner.ts:234` |

### Breaking changes that do NOT affect akmatori

- `session_switch` / `session_fork` extension events removed (0.65.0) — not used
- Session-replacement methods removed from `AgentSession` (0.65.0) — not used
- `session_directory` removed from extension/settings APIs (0.65.0) — `noExtensions: true` in akmatori
- Unknown single-dash CLI flags now error (0.65.0) — CLI mode only

### Automatic improvements (no code changes needed)

- **Prompt caching improvement** (0.67.4): Anthropic `cache_control` breakpoint added on last tool definition, improving cache hit rate for tool-heavy agent sessions → cost reduction
- **claude-opus-4-7 support** (0.67.4/0.67.5): Model available + adaptive thinking fixed for Opus 4.7
- **Stability**: `Container.render()` stack overflow fix for long sessions (0.67.1)
- **Stability**: Shutdown kills orphaned bash child processes on SIGTERM/SIGHUP (0.67.4)
- **Stability**: Auto-retry treats more transient errors as retryable (0.65.1, 0.66.0)
- **Stability**: `grep` broad-search no longer stalls when `context=0` (0.67.4)
- **Stability**: `find` tool gitignore isolation fixed + cancellation made abort-aware (0.67.4/0.67.6)
- **Stability**: Bash output truncation by line count now persists full output to temp file (0.65.1)
- **Fix**: RpcClient now forwards subprocess stderr to parent in real-time (0.65.1)

### New features worth adopting (require code changes)

1. **`defineTool()` helper (0.65.0)**: Wraps plain tool-object factories with full TypeScript parameter inference, eliminating the `as unknown as ToolDefinition` cast at `agent-runner.ts:308` and formalizing the return type of all `createXxxTool()` factories in `gateway-tools.ts`.

## Task 3: Adopt `defineTool()` for gateway tool factories (0.65.0)

This is a quality-of-life improvement — it tightens TypeScript inference and eliminates the `as unknown as ToolDefinition` cast. It does not change runtime behavior.

**Files:**
- Modify: `agent-worker/src/gateway-tools.ts`
- Modify: `agent-worker/src/agent-runner.ts` (remove the cast)

- [ ] **Step 1: Add defineTool import to gateway-tools.ts**

At the top of `agent-worker/src/gateway-tools.ts`, add the import alongside existing typebox imports:

```typescript
import { defineTool } from "@mariozechner/pi-coding-agent";
```

- [ ] **Step 2: Wrap createGatewayCallTool with defineTool**

Find `createGatewayCallTool` in `gateway-tools.ts`. The function currently returns a plain object. Wrap the return value:

```typescript
export function createGatewayCallTool(ctx: GatewayToolContext) {
  return defineTool({
    name: "gateway_call",
    label: "Gateway Call",
    description: "...",  // keep existing description
    parameters: GatewayCallParams,
    promptSnippet: "...",  // keep existing promptSnippet
    execute: async (args: GatewayCallInput) => {
      // keep existing execute body
    },
  });
}
```

Apply the same `defineTool(...)` wrap to the other four factory functions:
- `createListToolsForToolTypeTool`
- `createGetToolDetailTool`
- `createListToolTypesTool`
- `createExecuteScriptTool`

- [ ] **Step 3: Remove the cast in agent-runner.ts**

In `agent-runner.ts` around line 308, the bash tool cast reads:


The gateway tools now return properly typed `ToolDefinition` from `defineTool()`. Verify the bash tool cast is still needed (it may be — `createBashToolDefinition` returns specialized generic types). If the TypeScript compiler accepts it without the cast after the upgrade, remove it. If not, keep the cast for the bash tool only.

## What Was Evaluated But Not Adopted

| Feature | Reason skipped |
|---------|----------------|
| `AgentSessionRuntime` (0.65.0) | Akmatori doesn't switch sessions mid-run; existing `SessionManager.create/continueRecent` pattern is sufficient |
| `after_provider_response` hook (0.67.4/0.67.6) | No current need to inspect raw HTTP headers/status codes from providers |
| `--no-context-files` / `loadProjectContextFiles()` (0.67.4) | Akmatori uses `skillsOverride` in `DefaultResourceLoader`, not CLI flags |
| Multiple `--append-system-prompt` (0.67.2) | Akmatori injects system context via skills and bash guidelines, not CLI flags |
| `PI_CODING_AGENT=true` env detection (0.67.1) | Already set automatically by SDK startup; no action needed |
| OpenRouter routing support (0.67.1) | Not using OpenRouter routing features currently |
| `ctx.signal` in extension handlers (0.63.2) | `noExtensions: true` in akmatori's resource loader |
